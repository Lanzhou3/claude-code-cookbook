# 第21章 创建自定义技能

> 本章详细介绍如何创建和使用自定义技能（Skills），包括技能定义、提示词编写和参数处理。

---

## 21.1 技能概述

### 21.1.1 什么是技能

技能是 Claude Code 中的提示词模板系统，通过 Markdown 文件定义：

```
┌─────────────────────────────────────────────────────────────┐
│                      技能结构示意                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  skill-name/                                               │
│  └── SKILL.md                                              │
│                                                             │
│  ───────────────────────────────────────────────────────   │
│                                                             │
│  description: 代码审查技能              # 简短描述            │
│  allowed-tools: ["Bash", "Read"]    # 可用工具            │
│  argument-hint: "<files...>"          # 参数提示            │
│  when_to_use: 需要审查代码时            # 使用场景            │
│  model: sonnet                        # 使用模型            │
│                                                             │
│  # 技能标题                                                 │
│                                                             │
│  技能的具体指导内容...                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 21.1.2 技能 vs 插件

| 特性 | 技能 | 插件 |
|------|------|------|
| **本质** | Markdown 提示词模板 | Markdown + JSON 清单 |
| **编写难度** | 低 | 中 |
| **内容** | 提示词和工具配置 | 命令、技能、钩子、MCP/LSP |
| **适用场景** | 工作流程模板 | 功能包扩展 |

## 21.2 技能文件格式

### 21.2.1 SKILL.md 格式

技能通过 `SKILL.md` 文件定义：

```markdown
---
description: 自动化代码审查
allowed-tools: ["Bash", "Read", "Grep"]
argument-hint: "<files...>"
when_to_use: 当你需要对代码进行审查时
model: sonnet
---

# Code Review Skill

## 概述

这个技能帮助审查代码变更，检测潜在问题和代码质量。

## 审查项目

- 代码风格一致性
- 潜在 Bug
- 性能问题
- 安全漏洞
- 测试覆盖

## 使用方式

```
/skill code-review src/**/*.ts
```
```

### 21.2.2 Frontmatter 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `description` | string | 技能描述（用户可见） |
| `allowed-tools` | string[] | 允许使用的工具列表 |
| `argument-hint` | string | 参数提示，如 `<files...>` |
| `when_to_use` | string | 使用场景说明 |
| `model` | string | 使用的模型（如 `sonnet`、`opus`） |

## 21.3 技能目录结构

### 21.3.1 基本结构

```
~/.claude/skills/
└── custom/                    # 自定义技能目录
    └── my-skill/
        └── SKILL.md          # 技能定义文件
```

### 21.3.2 组织方式

技能通过目录名称命名，调用时使用 `plugin-name:skill-name` 格式（对于自定义技能直接用目录名）：

```
my-skill              # my-skill 技能（来自 ~/.claude/skills/my-skill/）
```

## 21.4 技能开发

### 21.4.1 创建技能目录

```bash
# 创建自定义技能目录
mkdir -p ~/.claude/skills/custom/my-skill

# 创建技能文件
cat > ~/.claude/skills/custom/my-skill/SKILL.md << 'EOF'
---
description: 我的自定义技能
allowed-tools: ["Bash", "Read"]
argument-hint: "<input>"
when_to_use: 当你需要执行特定任务时
model: sonnet
---

# 技能内容

这里描述技能的具体行为和指导。
EOF
```

### 21.4.2 技能开发最佳实践

```markdown
---
description: 高质量技能开发指南
allowed-tools: ["Bash", "Read", "Grep"]
---

# 技能开发最佳实践

## 清晰的角色定义
你是一位 [领域] 专家，擅长 [具体技能]。

## 明确的输出格式
请按以下格式输出：
1. 标题
2. 要点列表
3. 代码示例（如果适用）

## 约束条件
- 不要超过 500 字
- 只关注 [特定方面]
- 使用中文回答
```

## 21.5 内置技能参考

Claude Code 内置以下技能（位于 `src/skills/bundled/`）：

| 技能名称 | 说明 |
|---------|------|
| `simplify` | 简化复杂代码 |
| `debug` | 调试辅助 |
| `verify` | 验证代码正确性 |
| `batch` | 批量操作 |
| `stuck` | 解决卡顿问题 |
| `updateConfig` | 更新配置文件 |
| `keybindings` | 快捷键帮助 |
| `loremIpsum` | 生成占位文本 |
| `skillify` | 将提示词转换为技能 |
| `remember` | 记忆管理 |

**条件性技能（需要特定功能标志）：**

| 技能名称 | 条件 | 说明 |
|---------|------|------|
| `dream` | `KAIROS` | 梦想模式 |
| `hunter` | `REVIEW_ARTIFACT` | artifact 猎人 |
| `loop` | `AGENT_TRIGGERS` | 循环任务 |
| `scheduleRemoteAgents` | `AGENT_TRIGGERS_REMOTE` | 远程 Agent 调度 |
| `claudeApi` | `BUILDING_CLAUDE_APPS` | Claude API 构建 |
| `claudeInChrome` | 自动检测 | Chrome 中的 Claude |
| `runSkillGenerator` | `RUN_SKILL_GENERATOR` | 技能生成器 |
