# 第20章 插件系统

> 本章详细介绍 Claude Code 的插件系统，包括插件架构、插件清单格式、命令与技能开发、以及插件管理与发布。

---

## 20.1 插件系统概述

### 20.1.1 设计理念

Claude Code 的插件系统是**基于 Markdown 文件的声明式系统**，而非传统的编程接口。插件通过 `manifest.md` 清单文件和 Markdown 命令文件定义，无需编写 TypeScript 代码即可扩展功能。

```
┌─────────────────────────────────────────────────────────────┐
│                    插件系统架构                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   插件目录 ~/.claude/plugins/<plugin-name>/                  │
│   └── .claude-plugin/                                      │
│       ├── manifest.json    # 插件清单（名称、版本、组件）     │
│       ├── commands/       # 命令文件（Markdown + Frontmatter）│
│       ├── skills/         # 技能目录（SKILL.md 文件）        │
│       └── hooks/          # 钩子配置（JSON）                 │
│                                                             │
│   插件加载器                                                │
│   └── pluginLoader.ts → 加载清单                            │
│   └── loadPluginCommands.ts → 解析 Markdown 命令             │
│   └── loadPluginHooks.ts → 加载钩子配置                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 20.1.2 核心概念

| 概念 | 说明 |
|------|------|
| **manifest.json** | 插件清单文件，位于 `.claude-plugin/` 目录 |
| **命令 (Command)** | Markdown 文件，定义 slash 命令 |
| **技能 (Skill)** | 包含 `SKILL.md` 的目录 |
| **钩子 (Hook)** | 与用户钩子相同的 JSON 配置系统 |
| **MCP 服务器** | 通过 manifest 配置的 Model Context Protocol 服务器 |
| **LSP 服务器** | 通过 manifest 配置的 Language Server Protocol 服务器 |

### 20.1.3 插件 vs 技能 vs MCP

| 特性 | 插件 | 技能 | MCP |
|------|------|------|-----|
| **格式** | Markdown + JSON | Markdown 文件 | 外部进程 |
| **能力** | 命令、技能、钩子、MCP/LSP | Prompt 模板 | 提供工具 |
| **复杂度** | 中 | 低 | 中 |
| **适用场景** | 功能包扩展 | Prompt 优化 | 外部工具集成 |

## 20.2 插件目录结构

### 20.2.1 标准结构

```
~/.claude/plugins/my-plugin/
├── .claude-plugin/
│   ├── manifest.json      # 插件清单（必需）
│   └── hooks/
│       └── hooks.json     # 钩子配置（可选）
├── commands/               # 命令目录
│   ├── hello.md           # /hello 命令
│   └── git-status.md      # /git-status 命令
├── skills/                # 技能目录
│   └── code-review/
│       └── SKILL.md       # code-review 技能
├── agents/                # Agent 目录（可选）
│   └── reviewer.md
└── README.md             # 插件说明
```

### 20.2.2 manifest.json 结构

```json
// .claude-plugin/manifest.json
{
  "name": "my-plugin",
  "version": "1.0.0",
  "description": "我的插件描述",
  "author": {
    "name": "作者名称",
    "email": "author@example.com",
    "url": "https://github.com/author"
  },
  "repository": "https://github.com/author/claude-plugin-my-plugin",
  "license": "MIT",
  "keywords": ["git", "productivity"],

  // 可选：补充默认路径的额外路径
  "commands": ["./custom-commands/*.md"],
  "skills": ["./custom-skills/"],
  "hooks": {
    "PreToolUse": [...]
  },

  // 可选：MCP 服务器配置
  "mcpServers": {
    "my-server": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-filesystem", "/path"]
    }
  }
}
```

## 20.3 命令开发

### 20.3.1 命令文件格式

命令是带有 YAML Frontmatter 的 Markdown 文件：

```markdown
---
description: 显示 Git 仓库状态
allowed-tools: ["Bash", "Read"]
argument-hint: "[--short]"
arguments:
  - name: flags
    description: git status 附加参数
    required: false
when_to_use: 当你想快速查看当前 Git 仓库状态时
version: "1.0.0"
model: sonnet
effort: 1
shell: true
---

# Git Status 命令

这个命令显示当前 Git 仓库的详细状态信息。

## 功能

- 显示已修改的文件
- 显示暂存区的文件
- 显示未跟踪的文件
- 显示分支信息

## 使用示例

```
/git-status
/git-status --short
/git-status -s
```
```

### 20.3.2 Frontmatter 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `description` | string | 命令描述（用户可见） |
| `allowed-tools` | string[] | 允许使用的工具列表 |
| `argument-hint` | string | 参数提示，如 `[file]` |
| `arguments` | array | 参数定义 |
| `when_to_use` | string | 使用场景说明 |
| `version` | string | 命令版本 |
| `model` | string | 使用的模型（如 `sonnet`、`opus`） |
| `effort` | number/string | 任务难度（1-10 或 `low`/`medium`/`high`） |
| `shell` | boolean | 是否在 shell 中执行 |
| `user-invocable` | boolean | 是否可被用户调用（默认 true） |
| `disable-model-invocation` | boolean | 是否禁用模型调用 |

### 20.3.3 命令变量

命令内容中可使用以下变量：

| 变量 | 说明 |
|------|------|
| `${CLAUDE_PLUGIN_ROOT}` | 插件根目录路径 |
| `${CLAUDE_PLUGIN_DATA}` | 插件数据目录 |
| `${CLAUDE_SKILL_DIR}` | 当前技能目录（仅技能） |
| `${CLAUDE_SESSION_ID}` | 当前会话 ID |
| `${user_config.KEY}` | 用户配置的选项值 |
| `$ARGUMENTS` | 传递给命令的参数 |

### 20.3.4 命令路径配置

```json
// manifest.json - 使用额外命令路径
{
  "name": "my-plugin",
  "commands": [
    "./commands/*.md",           // glob 模式
    "./docs/guide.md"            // 单个文件
  ]
}
```

## 20.4 技能开发

### 20.4.1 技能目录结构

技能是包含 `SKILL.md` 文件的目录：

```
skills/
└── code-review/
    └── SKILL.md      # 技能定义文件
```

### 20.4.2 SKILL.md 格式

```markdown
---
description: 自动化代码审查
allowed-tools: ["Bash", "Read", "Grep"]
argument-hint: "<files...>"
when_to_use: 需要对代码变更进行审查时
model: sonnet
---

# Code Review Skill

## 概述

这个技能帮助审查代码变更，检测潜在问题和代码质量。

## 审查项目

- 代码风格一致性
- 潜在 Bug
- 性能问题
- 安全漏洞
- 测试覆盖

## 使用方式

```
/code-review src/**/*.ts
/code-review --full
```
```

### 20.4.3 技能命名

技能通过目录名称命名，调用时使用 `plugin-name:skill-name` 格式：

```
my-plugin:code-review    # my-plugin 插件的 code-review 技能
my-plugin:git-helper    # my-plugin 插件的 git-helper 技能
```

## 20.5 钩子配置

### 20.5.1 钩子文件位置

插件钩子可以在以下位置定义：

1. `.claude-plugin/hooks/hooks.json` - 外部钩子文件
2. `manifest.json` 中的 `hooks` 字段 - 内联钩子配置

### 20.5.2 hooks.json 格式

```json
// hooks/hooks.json
{
  "description": "自动格式化代码变更",
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "npm run format -- $ARGUMENTS",
            "if": "Write(*.ts)"
          }
        ]
      }
    ]
  }
}
```

### 20.5.3 manifest.json 内联钩子

```json
{
  "name": "my-plugin",
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "prompt",
            "prompt": "分析此命令是否安全: $ARGUMENTS。回答 safe 或 unsafe。",
            "model": "sonnet"
          }
        ]
      }
    ]
  }
}
```

## 20.7 用户配置选项

### 20.7.1 定义配置选项

插件可以在 manifest 中声明用户配置选项，用户在启用插件时被提示输入：

```json
{
  "name": "my-plugin",
  "userConfig": {
    "apiKey": {
      "type": "string",
      "title": "API Key",
      "description": "输入你的 API 密钥",
      "required": true,
      "sensitive": true
    },
    "maxRetries": {
      "type": "number",
      "title": "最大重试次数",
      "description": "API 请求失败时的最大重试次数",
      "default": 3,
      "min": 1,
      "max": 10
    }
  }
}
```

### 20.7.2 配置类型

| 类型 | 说明 |
|------|------|
| `string` | 字符串配置 |
| `number` | 数字配置 |
| `boolean` | 布尔配置 |
| `directory` | 目录路径 |
| `file` | 文件路径 |

### 20.7.3 在命令中使用配置

```markdown
---
description: 使用配置的 API 进行分析
---

# Analyzer

分析代码使用 ${user_config.apiKey} 进行认证。
最大重试次数: ${user_config.maxRetries}
```

## 20.8 插件市场

### 20.8.1 市场结构

插件市场是一个包含 `marketplace.json` 的 Git 仓库或目录：

```json
// marketplace.json
{
  "name": "my-marketplace",
  "owner": {
    "name": "Marketplace Owner",
    "url": "https://github.com/owner"
  },
  "plugins": [
    {
      "name": "plugin-one",
      "source": {
        "source": "github",
        "repo": "owner/plugin-one"
      },
      "description": "插件一描述"
    },
    {
      "name": "plugin-two",
      "source": {
        "source": "npm",
        "package": "@owner/plugin-two"
      }
    }
  ]
}
```

### 20.8.2 插件来源类型

| 来源 | 配置示例 |
|------|---------|
| **GitHub** | `{ "source": "github", "repo": "owner/repo" }` |
| **Git URL** | `{ "source": "git", "url": "https://..." }` |
| **npm** | `{ "source": "npm", "package": "@org/package" }` |
| **pip** | `{ "source": "pip", "package": "package-name" }` |
| **本地路径** | `"./local-plugin"` |

### 20.8.3 注册市场

在 `settings.json` 中注册插件市场：

```json
{
  "plugins": {
    "marketplaces": {
      "my-marketplace": {
        "source": "github",
        "repo": "owner/claude-plugins"
      }
    }
  }
}
```

## 20.9 插件管理命令

### 20.9.1 插件命令

| 命令 | 说明 |
|------|------|
| `/plugins` | 打开插件管理器 |
| `/plugins install <name>` | 安装插件 |
| `/plugins uninstall <name>` | 卸载插件 |
| `/plugins enable <name>` | 启用插件 |
| `/plugins disable <name>` | 禁用插件 |
| `/plugins update` | 更新所有插件 |
| `/reload-plugins` | 重新加载插件 |

### 20.9.2 热重载

修改插件文件后，使用 `/reload-plugins` 无需重启即可加载更改。

## 20.10 内置插件

### 20.10.1 内置插件系统

Claude Code 提供内置插件系统，用于官方功能扩展：

```typescript
// 内置插件注册 (plugins/builtinPlugins.ts)
import { registerBuiltinPlugin } from '...'

registerBuiltinPlugin({
  name: 'official-tool',
  description: '官方工具描述',
  defaultEnabled: true,
  skills: [...],
  hooks: {...},
  mcpServers: {...}
})
```

### 20.10.2 内置插件标识

内置插件 ID 使用 `@builtin` 后缀：

```
official-tool@builtin
```

## 20.11 依赖管理

### 20.11.1 声明依赖

```json
{
  "name": "dependent-plugin",
  "dependencies": ["base-plugin@my-marketplace"]
}
```

### 20.11.2 依赖解析

依赖会在插件加载时自动解析：
- 如果依赖未启用，提示用户启用
- 如果依赖未安装，自动安装（如果可从同一市场获取）
