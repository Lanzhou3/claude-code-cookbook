## 23.1 开发环境

| 要求 | 版本 | 说明 |


```bash

bun install

# 验证安装
bun run src/entrypoints/cli.tsx --version
```

## 23.3 开发工作流

### 23.3.1 Git 工作流

```
┌─────────────────────────────────────────────────────────────┐
│                    Git 工作流                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 创建特性分支                                            │
│     main ────────────────────────────────────────          │
│           │                                                │
│           ├── feature/my-feature ──→ PR ──→ main            │
│                                                             │
│  2. 提交规范                                                │
│     <type>: <description>                                   │
│                                                             │
│     feat: 添加新工具                                        │
│     fix: 修复权限检查                                        │
│     docs: 更新文档                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 23.3.2 提交规范

```bash
# 格式
<type>(<scope>): <description>

# 示例
git commit -m "feat(tools): 添加 GitStatusTool"
git commit -m "fix(mcp): 修复 stdio 传输断开连接"
git commit -m "docs(readme): 更新安装说明"
```

### 23.3.3 类型规范

| 类型 | 说明 | 何时使用 |
|------|------|----------|
| **feat** | 新功能 | 添加新功能 |
| **fix** | Bug 修复 | 修复问题 |
| **docs** | 文档 | 文档更新 |
| **style** | 格式 | 不影响逻辑 |
| **refactor** | 重构 | 代码重构 |
| **test** | 测试 | 添加测试 |
| **chore** | 杂项 | 构建/工具 |

## 23.4 添加新工具

### 23.4.1 创建工具目录

```bash
# 创建目录结构
mkdir -p src/tools/MyNewTool
```

### 23.4.2 实现工具

```typescript
// src/tools/MyNewTool/index.ts
import { z } from 'zod'
import type { Tool, ToolResult, ToolUseContext } from '../Tool.js'
import type { CanUseToolFn } from '../hooks/useCanUseTool.js'
import type { AssistantMessage } from '../types/message.js'

// 注意：工具是工厂函数，返回 Tool 对象，不是类实现
export const MyNewTool = (): Tool => ({
  name: 'MyNew',
  inputSchema: z.object({
    param1: z.string().describe('参数1'),
    param2: z.number().optional().describe('可选参数2'),
  }),

  // 注意：是 call 方法，不是 execute
  async call(
    args: z.infer<ReturnType<typeof MyNewTool>['inputSchema']>,
    context: ToolUseContext,
    canUseTool: CanUseToolFn,
    parentMessage: AssistantMessage,
  ): Promise<ToolResult<string>> {
    try {
      // 执行逻辑
      const result = doSomething(args.param1, args.param2)

      // 注意：返回 data 字段，不是 content
      return {
        data: JSON.stringify(result),
      }
    } catch {
      return {
        data: '',
      }
    }
  },

  description() {
    return '我的新工具'
  },

  isEnabled() {
    return true
  },

  isReadOnly() {
    return true
  },

  isConcurrencySafe() {
    return true
  },
})
```

### 23.4.3 注册工具

自定义工具通过 MCP 服务器或插件系统集成，而非直接修改 tools.ts。工具系统使用**动态条件加载**，通过 `feature()` 标志和环境变量控制工具的可用性。

如需添加新内置工具，请在 `tools.ts` 的 `getAllBaseTools()` 函数中按现有模式条件添加。

## 23.5 测试

### 23.5.1 编写测试

```typescript
// src/tools/MyNewTool/MyNewTool.test.ts
import { describe, it, expect } from 'bun:test'
import { MyNewTool } from './index.js'
import { z } from 'zod'

describe('MyNewTool', () => {
  const tool = MyNewTool()
  const schema = tool.inputSchema

  it('should validate correct input with Zod', () => {
    const result = schema.safeParse({ param1: 'test' })
    expect(result.success).toBe(true)
  })

  it('should reject invalid input with Zod', () => {
    const result = schema.safeParse({})
    expect(result.success).toBe(false)
  })

  it('should have correct structure', () => {
    expect(tool.name).toBe('MyNew')
    expect(tool.isEnabled()).toBe(true)
    expect(tool.isReadOnly?.({ param1: 'test' })).toBe(true)
  })
})
```

### 23.5.2 运行测试

```bash
# 运行所有测试
bun test

# 运行单个文件
bun test src/tools/MyNewTool/MyNewTool.test.ts

# 带覆盖率
bun test --coverage
```

### 23.6.2 提交 PR

```bash
# 1. 推送分支
git push origin feature/my-feature

# 2. 创建 PR
gh pr create \
  --title "feat(tools): 添加新工具" \
  --body "$(cat <<'EOF'
## 描述
添加新的 MyNewTool 工具

## 类型
- [ ] feat
- [ ] fix
- [ ] docs

## 测试
- [ ] 本地测试通过
- [ ] 测试用例添加

## 截图（如果适用）
EOF
)"
```

## 23.7 二次开发

### 23.7.1 常见二次开发场景

| 场景 | 说明 | 修改点 |
|------|------|--------|
| **自定义工具** | 添加项目特定工具 | `tools/` |
| **自定义命令** | 添加项目特定命令 | `commands/` |
| **UI 修改** | 定制界面风格 | `ink/`, `components/` |
| **API 修改** | 支持新 API | `services/api/` |
| **协议扩展** | 支持新协议 | `services/mcp/` |

### 23.7.2 保持同步

```bash
# 1. 获取上游更新
git fetch upstream

# 2. 合并到本地 main
git checkout main
git merge upstream/main

# 3. 合并到特性分支
git checkout feature/my-feature
git merge main

# 4. 解决冲突
# ...
```

### 23.7.3 版本管理

```bash
# 使用 semver
git tag v1.0.0

# 创建发布分支
git checkout -b release/v1.1.0 main

# 推送标签
git push origin v1.1.0
```
