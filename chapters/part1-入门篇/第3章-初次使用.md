### 3.1.2 命令行参数启动

你也可以直接附带问题启动：

```bash
# 直接问问题（启动后自动发送）

claude "解释这段代码" --model opus

# 以安静模式启动（不显示欢迎界面）
claude "检查代码" --quiet
```

## 3.2 基本交互

### 3.2.1 发送消息

在 REPL 界面中，直接输入你的问题或指令：

```
> 帮我看看当前目录有什么文件

当前目录包含以下文件和文件夹：

📁 src/           - 源代码目录
📁 tests/         - 测试文件目录
📄 package.json   - Node.js 项目配置
📄 README.md      - 项目说明文档
📄 .gitignore     - Git 忽略配置
```

### 3.2.2 多轮对话

Claude Code 支持多轮对话，可以在同一会话中连续交互：

```
> 帮我创建一个新文件

好的，请问您想创建什么类型的文件？
例如：Python 脚本、JavaScript 模块、配置文件等。

> 一个 Python 脚本来处理 JSON 数据

明白，我来创建一个 Python 脚本来处理 JSON 数据。
...
```

### 3.2.3 取消操作

如果 Claude 正在执行耗时操作，你可以：

- 按 `Ctrl+C` 取消当前操作
- 按 `Ctrl+L` 清屏
- 输入 `/abort` 中止当前任务

## 3.3 理解工具调用

Claude Code 的强大之处在于它可以调用各种工具来完成任务。

### 3.3.1 工具调用示例

当你请求查看文件时：

```
> 读取 src/index.ts 的内容
```

Claude Code 会调用 `FileReadTool`，你会在界面上看到：

```
[TOOL_CALL] FileReadTool
  path: src/index.ts

[RESULT]
---
1 | import { main } from './main'
2 | import { config } from './config'
3 |
4 | console.log('Hello, World!')
5 |
6 | export { main }
---
```

### 3.3.2 常用工具调用

| 工具 | 用途 | 示例 |
|------|------|------|
| **BashTool** | 执行 Shell 命令 | `npm install`、`git status` |
| **FileReadTool** | 读取文件内容 | 查看源码、配置文件 |
| **FileEditTool** | 编辑文件 | 修改代码、添加注释 |
| **FileWriteTool** | 写入新文件 | 创建新模块 |
| **GlobTool** | 模式匹配文件 | 找所有 `.ts` 文件 |
| **GrepTool** | 搜索文件内容 | 找特定函数调用 |
| **AgentTool** | 启动子代理 | 并行执行复杂任务 |

### 3.3.3 工具输出格式

Claude Code 会以结构化方式展示工具结果：

```
📋 文件读取: src/utils/helper.ts
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 1│ export function formatDate(date: Date): string {
 2│   return date.toISOString().split('T')[0];
 3│ }
 4│
 5│ export function parseJSON<T>(json: string): T {
 6│   return JSON.parse(json);
 7│ }
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✨ 已读取 7 行代码
```

## 3.4 权限确认

### 3.4.1 权限提示

当你请求可能敏感的操作时，Claude Code 会请求确认：

```
> 运行 rm -rf /tmp/test 删除临时文件

⚠️  此操作涉及删除文件，请确认：

  命令: rm -rf /tmp/test
  影响: 将删除 /tmp/test 目录及其所有内容

  [y] 确认执行  [n] 拒绝  [a] 允许所有类似操作
```

### 3.4.2 权限模式

根据你的权限模式，确认行为会有所不同：

| 模式 | 确认频率 | 说明 |
|------|----------|------|
| **default** | 每次敏感操作前 | 安全但频繁打扰 |
| **plan** | 只在计划阶段 | 计划确认后自动执行 |
| **acceptEdits** | 不确认编辑 | 自动接受文件修改 |
| **auto** | 完全自动 | 无需任何确认 |

## 3.5 会话管理

### 3.5.1 退出和恢复

```
> /exit

感谢使用 Claude Code！会话已保存。
下次可以通过 /resume 恢复继续。
```

### 3.5.2 恢复会话

```bash
# 恢复上次会话
claude --resume

# 或者在启动后输入
> /resume
```

Claude Code 会加载上次会话的上下文，让你继续之前的工作。

### 3.5.3 会话历史

```
> /history

最近会话：
  1. 今天 14:30 - "帮我审查代码" - 进行中
  2. 今天 10:15 - "创建新组件"  - 已完成
  3. 昨天 16:42 - "修复 bug"    - 已完成

> /history 2
# 恢复编号为 2 的会话
```

## 3.6 常用命令

### 3.6.1 帮助命令

```bash
> /help

可用命令：
──────────────────────────────────────
  /help [topic]    查看帮助
  /exit            退出 Claude Code
  /resume          恢复上次会话
  /history         查看会话历史
  /clear           清屏
  /model [name]    切换模型
──────────────────────────────────────

> /help model
查看当前可用模型：
  - claude-sonnet-4-6 (默认)
  - claude-opus-4-6
  - claude-haiku-4-5
```

### 3.6.2 Git 操作

```bash
> /git status
# 查看当前 Git 状态

> /git commit -m "fix: 修复登录bug"
# 执行 Git 提交

> /git push
# 推送到远程仓库
```

### 3.6.3 模型切换

```bash
> /model opus
# 切换到 Opus 模型（更强推理）

> /model haiku
# 切换到 Haiku 模型（更快响应）
```

## 3.7 实用技巧

### 3.7.1 自动补全

在输入过程中，可以：

- 按 `Tab` 自动补全命令
- 按 `↑/↓` 浏览历史输入
- 按 `Ctrl+R` 搜索历史

### 3.7.2 多行输入

输入多行内容时，使用 `\` 续行：

```
> 写一个函数，接受三个参数 \
  第一个是用户名 \
  第二个是邮箱 \
  返回用户对象
```

或者使用粘贴模式（直接粘贴代码块）。

### 3.7.3 快捷操作

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+C` | 取消当前操作 |
| `Ctrl+L` | 清屏 |
| `Ctrl+D` | 退出 Claude Code |
| `Tab` | 自动补全 |
| `↑/↓` | 浏览历史 |

## 3.8 常见问题

### 3.8.1 首次使用问题

**Q: 启动后没有任何反应**

```bash
# 检查安装
claude --version

# 检查 API Key
echo $ANTHROPIC_API_KEY

# 查看详细错误
claude --debug
```

**Q: 工具调用没有响应**

可能是工具执行超时或出错，可以：
- 查看详细日志：`claude --verbose`
- 尝试单个工具调用
- 检查网络连接

### 3.8.2 交互问题

**Q: 如何中断长时间运行的任务？**

按 `Ctrl+C` 可以中断当前任务。

**Q: 会话可以保存多久？**

会话默认保存 30 天。你可以使用 `/export` 导出会话记录。
