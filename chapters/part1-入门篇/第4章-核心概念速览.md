## 4.1 消息与会话模型

### 4.1.1 消息结构

Claude Code 使用结构化消息进行通信：

```typescript
interface Message {
  role: 'user' | 'assistant' | 'system';
  content: string | ContentBlock[];
  metadata?: {
    timestamp: number;
    model?: string;
    usage?: TokenUsage;
  };
}
```

### 4.1.2 对话流程

```
┌─────────────────────────────────────────────────────────────┐
│                      用户消息                                 │
│                  "帮我创建一个函数"                            │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                    上下文构建                                 │
│  - 加载系统提示                                               │
│  - 加载会话历史                                               │
│  - 加载项目上下文                                             │
│  - 加载工具定义                                               │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                    AI 推理                                   │
│              Claude API 生成响应                              │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                   响应渲染                                   │
│           格式化输出 + 工具调用（如果需要）                   │
└─────────────────────────────────────────────────────────────┘
```

### 4.1.3 Token 与成本

Claude Code 会跟踪每次对话的 token 使用：

```bash
> 查看 token 使用

┌────────────────────────────────────────┐
│  当前会话 Token 使用                    │
├────────────────────────────────────────┤
│  模型: Claude Sonnet 4                  │
│  输入 tokens:  1,234                    │
│  输出 tokens:    567                    │
│  总计:         1,801                    │
│  估算成本:     $0.008                   │
└────────────────────────────────────────┘
```

## 4.2 工具系统

### 4.2.1 工具定义

每个工具都有标准的定义结构：

```typescript
interface Tool {
  name: string;                    // 工具名称
  description: string;             // 工具描述
  inputSchema: JSONSchema;         // 输入参数模式
  outputSchema?: JSONSchema;        // 输出模式（可选）
}
```

### 4.2.2 工具分类

Claude Code 的工具采用动态加载机制，工具数量取决于 feature flags 和配置：

| 类别 | 始终可用 | 条件可用（示例） |
|------|----------|-------------------|
| **文件操作** | FileReadTool, FileEditTool, FileWriteTool, NotebookEditTool | - |
| **搜索** | GlobTool*, GrepTool* | - (* 无内置搜索时) |
| **Shell** | BashTool | PowerShellTool |
| **智能代理** | AgentTool, TaskOutputTool, TaskStopTool | TeamCreateTool, TeamDeleteTool |
| **任务管理** | TodoWriteTool | TaskCreateTool, TaskGetTool, TaskUpdateTool, TaskListTool |
| **用户交互** | AskUserQuestionTool, BriefTool | - |
| **Web** | WebFetchTool, WebSearchTool | WebBrowserTool |
| **计划/模式** | EnterPlanModeTool, ExitPlanModeV2Tool | - |
| **MCP** | ListMcpResourcesTool, ReadMcpResourceTool | - |
| **定时任务** | - | CronCreateTool, CronDeleteTool, CronListTool |

### 4.2.3 工具执行流程

```
用户请求
    │
    ▼
工具选择 ──→ [无工具] ──→ 直接回复
    │
    │
工具调度
    │
    ├─→ [并发工具] ──→ 并行执行 ──→ 聚合结果
    │
    └─→ [顺序工具] ──→ 依次执行 ──→ 组合结果
    │
    ▼
结果处理 ──→ 格式化输出
```

### 4.2.4 工具并发策略

Claude Code 使用分区并发（Partitioned Concurrency）来优化工具执行：

```typescript
// 独立的工具调用可以并行执行
const results = await Promise.all([
  readFile('src/a.ts'),
  readFile('src/b.ts'),
  glob('src/**/*.ts')
]);

// 相关的工具调用按顺序执行
const content = await readFile('config.json');
const config = JSON.parse(content);
await validateConfig(config);
```

## 4.3 上下文管理

### 4.3.1 上下文分层

Claude Code 的上下文分为四层：

```
┌─────────────────────────────────────────────────────────────┐
│                    系统级上下文                              │
│         - 内置工具定义                                       │
│         - 系统提示词                                         │
│         - 默认行为规则                                       │
├─────────────────────────────────────────────────────────────┤
│                    项目级上下文                              │
│         - .claude/ 配置                                     │
│         - 项目结构                                           │
│         - 语言/框架检测                                       │
├─────────────────────────────────────────────────────────────┤
│                    会话级上下文                              │
│         - 对话历史                                           │
│         - 当前工作目录                                       │
│         - 用户偏好设置                                       │
├─────────────────────────────────────────────────────────────┤
│                    请求级上下文                              │
│         - 当前用户输入                                        │
│         - 最近的工具结果                                      │
│         - 临时状态                                           │
└─────────────────────────────────────────────────────────────┘
```

### 4.3.2 上下文窗口

Claude Code 使用智能上下文管理来优化 token 使用：

| 模型 | 上下文窗口 | Claude Code 优化 |
|------|------------|-----------------|
| Sonnet 4 | 200K | 自动摘要、长文件截断 |
| Opus 4 | 200K | 同上 |
| Haiku 4 | 200K | 同上 |

### 4.3.3 上下文压缩

当上下文接近限制时，Claude Code 会自动压缩：

1. **摘要历史** - 将早期对话转换为摘要
2. **精简工具输出** - 保留关键信息，删除冗余
3. **文件路径压缩** - 使用相对路径代替绝对路径

## 4.4 权限系统

### 4.4.1 权限模型

```
┌─────────────────────────────────────────────────────────────┐
│                      权限检查引擎                            │
├─────────────────────────────────────────────────────────────┤
│  输入 ──→ 规则匹配 ──→ 决策 ──→ 执行/拒绝                    │
│           │                                                 │
│           ▼                                                 │
│        风险评估                                              │
│           │                                                 │
│           ▼                                                 │
│        用户确认（如需要）                                     │
└─────────────────────────────────────────────────────────────┘
```

### 4.4.2 权限规则

```typescript
// 权限规则示例
const permissionRules = [
  {
    action: 'file:write',
    path: '**/*.ts',
    requireConfirm: false,
    risk: 'low'
  },
  {
    action: 'file:delete',
    path: '**/node_modules/**',
    requireConfirm: true,
    risk: 'high'
  },
  {
    action: 'bash:exec',
    command: 'rm -rf *',
    requireConfirm: true,
    risk: 'critical'
  }
];
```

### 4.4.3 风险级别

| 级别 | 说明 | 默认行为 |
|------|------|----------|
| **low** | 安全操作，如读取文件 | 自动执行 |
| **medium** | 可能有风险，如写入文件 | 确认后执行 |
| **high** | 危险操作，如删除文件 | 明确确认 |
| **critical** | 破坏性操作，如系统命令 | 拒绝执行 |

## 4.5 扩展机制

### 4.5.1 三大扩展点

Claude Code 提供三个扩展机制：

| 扩展点 | 用途 | 配置位置 |
|--------|------|----------|
| **Plugins** | 功能插件扩展 | `~/.claude/plugins/` |
| **Skills** | 提示词模板库 | `~/.claude/skills/` |
| **MCP Servers** | 外部工具服务 | `~/.claude/settings.json` |

### 4.5.2 插件系统

```
~/.claude/plugins/
├── my-plugin/
│   ├── index.ts          # 插件入口
│   ├── commands/         # 自定义命令
│   ├── tools/            # 自定义工具
│   └── hooks/            # 生命周期钩子
└── plugin.json           # 插件配置
```

### 4.5.3 技能系统

```
~/.claude/skills/
├── bundled/              # 内置技能
│   ├── code-review/
│   └── refactor/
└── custom/               # 自定义技能
    └── my-skill/
        └── skill.md
```

### 4.5.4 MCP 集成

```json
// ~/.claude/settings.json
{
  "mcpServers": [
    {
      "name": "filesystem",
      "command": "npx",
      "args": ["@modelcontextprotocol/server-filesystem", "/path"]
    }
  ]
}
```

## 4.6 状态管理

### 4.6.1 双层状态架构

```
┌─────────────────────────────────────────────────────────────┐
│                   Bootstrap State                           │
│                      (单例模式)                              │
├─────────────────────────────────────────────────────────────┤
│  - sessionId        : 当前会话 ID                            │
│  - model           : 当前使用的模型                          │
│  - apiKey          : API 密钥                                │
│  - costTracker     : 成本追踪器                              │
│  - metrics         : 性能指标                                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   AppStateStore                             │
│                    (响应式存储)                               │
├─────────────────────────────────────────────────────────────┤
│  - messages        : 对话消息列表                            │
│  - currentTask     : 当前任务                                │
│  - uiState        : UI 状态                                  │
│  - toolResults    : 工具执行结果                            │
└─────────────────────────────────────────────────────────────┘
```

### 4.6.2 状态更新流程

```
用户操作 ──→ Action ──→ Reducer ──→ 新状态 ──→ 触发 UI 更新
                            │
                            ▼
                         Middleware
                            │
                            ▼
                     持久化（如需要）
```

## 4.7 MCP 协议

### 4.7.1 MCP 架构

```
┌──────────────────┐         ┌──────────────────┐
│   Claude Code    │ ←─────→ │   MCP Server     │
│                  │  JSON-RPC              │
│   (MCP Client)   │         │                  │
└──────────────────┘         └──────────────────┘
        │                              │
        │                              ▼
        │                    ┌──────────────────┐
        │                    │   Local Tools    │
        │                    │   Resources     │
        │                    │   Prompts       │
        │                    └──────────────────┘
        │
        ▼
┌──────────────────┐
│   Claude API     │
│                  │
└──────────────────┘
```

### 4.7.2 传输类型

| 传输类型 | 说明 | 使用场景 |
|----------|------|----------|
| **stdio** | 标准输入输出 | 本地 MCP 服务器 |
| **SSE** | Server-Sent Events | HTTP 长连接 |
| **HTTP** | 轮询方式 | 简单集成 |
| **WebSocket** | 双向通信 | 实时应用 |

### 4.7.3 MCP 工具调用

```
用户请求
    │
    ▼
Claude Code 检测到 MCP 工具需求
    │
    ▼
通过 MCP 协议发送请求
    │
    ▼
MCP Server 执行工具
    │
    ▼
返回结果给 Claude Code
    │
    ▼
结果整合到响应中
```

## 4.8 本章小结
