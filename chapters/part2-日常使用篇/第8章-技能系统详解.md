# 第8章 技能系统详解

> 本章详细介绍 Claude Code 的技能（Skills）系统，包括内置技能的使用方法、自定义技能的创建以及技能工作流。

---

## 8.1 技能概述

### 8.1.1 什么是技能

技能（Skills）是 Claude Code 中的提示词模板系统，通过 Markdown 文件定义并通过目录组织。技能封装了特定任务的完整工作流程，包含：

- **系统提示** - 指导 AI 如何执行任务
- **工具配置** - 指定可用的工具集
- **触发条件** - 定义何时激活技能

### 8.1.2 技能与命令的区别

| 特性 | 技能 (Skills) | 命令 (Commands) |
|------|--------------|----------------|
| 本质 | Markdown 提示词模板 | slash 命令 |
| 触发 | `/skill name` 或自然语言 | `/command` |
| 灵活性 | 可传参数 | 固定参数 |
| 用途 | 复杂任务模板 | 简单操作 |

### 8.1.3 技能文件格式

技能通过 `SKILL.md` 文件定义，位于特定目录下：

```
~/.claude/skills/my-skill/
└── SKILL.md          # 技能定义文件
```

## 8.2 内置技能

### 8.2.1 查看可用技能

内置技能位于 `src/skills/bundled/` 目录，在 Claude Code 启动时自动加载。

**核心内置技能（始终可用）：**

| 技能名称 | 说明 |
|---------|------|
| `simplify` | 简化复杂代码 |
| `debug` | 调试辅助 |
| `verify` | 验证代码正确性 |
| `batch` | 批量操作 |
| `stuck` | 解决卡顿问题 |
| `updateConfig` | 更新配置文件 |
| `keybindings` | 快捷键帮助 |
| `loremIpsum` | 生成占位文本 |
| `skillify` | 将提示词转换为技能 |
| `remember` | 记忆管理 |

**条件性技能（需启用特定功能标志）：**

| 技能名称 | 条件 | 说明 |
|---------|------|------|
| `dream` | `KAIROS` 或 `KAIROS_DREAM` | 梦想模式 |
| `hunter` | `REVIEW_ARTIFACT` | artifact 猎人 |
| `loop` | `AGENT_TRIGGERS` | 循环任务 |
| `scheduleRemoteAgents` | `AGENT_TRIGGERS_REMOTE` | 远程 Agent 调度 |
| `claudeApi` | `BUILDING_CLAUDE_APPS` | Claude API 构建 |
| `claudeInChrome` | 自动检测 | Chrome 中的 Claude |
| `runSkillGenerator` | `RUN_SKILL_GENERATOR` | 技能生成器 |

### 8.2.2 simplify 技能

简化复杂代码，将其转换为更易读的版本：

```bash
# 使用 simplify 技能
> /skill simplify

# 指定文件
> /skill simplify src/complex-algorithm.ts
```

### 8.2.3 debug 技能

帮助调试代码问题：

```bash
# 激活调试技能
> /skill debug

# 分析特定错误
> /skill debug --error "Cannot read property 'x' of undefined"
```

### 8.2.4 verify 技能

验证代码正确性和完整性：

```bash
# 验证代码
> /skill verify src/

# 验证特定功能
> /skill verify --coverage 80
```

### 8.2.5 stuck 技能

当开发过程卡顿时提供帮助：

```bash
# 解决卡顿
> /skill stuck

# 描述问题
> /skill stuck --problem "不确定如何重构这个模块"
```

### 8.2.6 updateConfig 技能

帮助更新 Claude Code 配置文件：

```bash
# 更新配置
> /skill updateConfig

# 指定配置项
> /skill updateConfig --setting "permissions"
```

## 8.4 自定义技能

### 8.4.1 创建技能目录

```bash
# 创建自定义技能目录
mkdir -p ~/.claude/skills/custom/my-skill

# 创建技能文件
cat > ~/.claude/skills/custom/my-skill/SKILL.md << 'EOF'
---
description: 我的自定义技能
allowed-tools: ["Bash", "Read", "Grep"]
argument-hint: "<input>"
when_to_use: 当你需要执行特定任务时
model: sonnet
---

# 技能内容

这里描述技能的具体行为和指导。
EOF
```

### 8.4.2 SKILL.md 格式

```markdown
---
description: 技能描述
allowed-tools: ["Bash", "Read"]
argument-hint: "<args>"
when_to_use: 使用场景说明
model: sonnet
---

# 技能标题

技能的具体指导内容...
```

### 8.4.3 技能字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `description` | string | 技能描述（用户可见） |
| `allowed-tools` | string[] | 允许使用的工具列表 |
| `argument-hint` | string | 参数提示 |
| `when_to_use` | string | 使用场景说明 |
| `model` | string | 使用的模型 |

### 8.4.4 加载自定义技能

```bash
# 重启 Claude Code 加载新技能
/exit
claude

# 或者使用 reload 命令
> /reload skills
```

## 8.5 技能开发

### 8.5.1 技能目录结构

```
my-skill/
├── SKILL.md           # 技能定义（必需）
├── prompts/           # 提示词片段（可选）
│   ├── header.md
│   └── footer.md
└── templates/         # 输出模板（可选）
    └── report.md
```

### 8.5.2 技能开发最佳实践

```markdown
---
description: 高质量技能开发指南
allowed-tools: ["Bash", "Read", "Grep"]
---

# 技能开发最佳实践

## 清晰的角色定义
你是一位 [领域] 专家，擅长 [具体技能]。

## 明确的输出格式
请按以下格式输出：
1. 标题
2. 要点列表
3. 代码示例（如果适用）

## 约束条件
- 不要超过 500 字
- 只关注 [特定方面]
- 使用中文回答
```
