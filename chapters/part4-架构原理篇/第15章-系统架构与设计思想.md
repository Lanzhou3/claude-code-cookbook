## 15.3 核心模块设计

### 15.3.1 QueryEngine - 查询引擎

QueryEngine 是 Claude Code 的核心，负责管理整个对话生命周期：

```typescript
// QueryEngine 核心职责
class QueryEngine {
  // 配置管理
  private config: QueryEngineConfig

  // 会话状态
  private mutableMessages: Message[]
  private abortController: AbortController
  private totalUsage: TokenUsage

  // 核心方法：异步生成器模式
  async *submitMessage(
    messages: Message[],
    context: Context
  ): AsyncGenerator<SubmitMessageResult>
}
```

**设计亮点**：
- **异步生成器模式**：使用 `AsyncGenerator` 实现流式响应，边生成边输出
- **可中止**：支持 `AbortController`，用户可随时中断长时间操作
- **状态累积**：跟踪 `totalUsage`，实时显示 token 消耗

### 15.3.2 工具系统架构

工具系统采用**接口-实现分离+动态加载**的设计：

```typescript
// 工具接口定义 (Tool.ts) - 简化版
// 实际 Tool 是一个复杂的泛型类型
type Tool<
  Input extends AnyObject = AnyObject,
  Output = unknown,
> = {
  readonly name: string              // 工具唯一名称
  readonly inputSchema: Input       // 输入参数模式
  call(args: Input, context: ToolUseContext, ...): Promise<ToolResult<Output>>
  description(input: Input, options: {...}): Promise<string>
  validateInput?(input: Input, context: ToolUseContext): Promise<ValidationResult>
  isEnabled(): boolean
  // ... 还有许多可选属性
}

// 工具注册机制 (tools.ts)
// 核心函数：getAllBaseTools() - 返回所有可能可用的工具
export function getAllBaseTools(): Tools {
  return [
    AgentTool,
    TaskOutputTool,
    BashTool,
    ...(hasEmbeddedSearchTools() ? [] : [GlobTool, GrepTool]),
    ExitPlanModeV2Tool,
    FileReadTool,
    FileEditTool,
    FileWriteTool,
    // ... 更多工具，有些是条件加载的
  ]
}
```

**工具分类**（基于实际源码）：

| 类别 | 始终包含 | 条件包含（示例） |
|------|----------|------------------|
| **文件操作** | FileReadTool, FileEditTool, FileWriteTool, NotebookEditTool | - |
| **搜索** | GlobTool（无内置搜索时）, GrepTool（无内置搜索时） | - |
| **Shell** | BashTool | PowerShellTool（启用时） |
| **智能代理** | AgentTool, TaskOutputTool, TaskStopTool | TeamCreateTool, TeamDeleteTool（swarm启用时） |
| **用户交互** | AskUserQuestionTool, BriefTool | - |
| **任务管理** | TodoWriteTool | TaskCreateTool, TaskGetTool, TaskUpdateTool, TaskListTool（TodoV2启用时） |
| **Web** | WebFetchTool, WebSearchTool | WebBrowserTool（feature flag启用时） |
| **计划/模式** | ExitPlanModeV2Tool, EnterPlanModeTool | - |
| **MCP** | ListMcpResourcesTool, ReadMcpResourceTool | - |
| **技能** | SkillTool | - |
| **定时任务** | - | CronCreateTool, CronDeleteTool, CronListTool（AGENT_TRIGGERS启用时） |
| **其他** | - | SleepTool, MonitorTool, SendUserFileTool, PushNotificationTool等 |

**重要特性**：
- **工具数量动态变化**：取决于 feature flags、环境变量、构建类型
- **无固定数字**：不能简单说"44个工具"，实际范围约20-40+
- **条件加载**：通过 `feature('FEATURE_NAME')` 和环境变量控制

### 15.3.3 状态管理

Claude Code 采用**双层状态架构**：

```
┌─────────────────────────────────────────────────────────────┐
│                    STATE 模块 (单例)                          │
│                    bootstrap/state.ts                        │
├─────────────────────────────────────────────────────────────┤
│  导出内容：                                                  │
│  - const STATE: State  (状态对象)                           │
│  - function getSessionId(): SessionId                        │
│  - function getCwdState(): string                           │
│  - function setCwdState(cwd: string): void                  │
│  - function addToTotalCostState(...): void                   │
│  - ... 更多函数                                              │
├─────────────────────────────────────────────────────────────┤
│  State 类型包含80+字段：                                     │
│  - sessionId, parentSessionId                               │
│  - originalCwd, projectRoot, cwd                           │
│  - totalCostUSD, totalAPIDuration                          │
│  - modelUsage, metrics                                     │
│  - ... 等等                                                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   AppStateStore (响应式)                    │
│                   state/AppStateStore.ts                     │
├─────────────────────────────────────────────────────────────┤
│  - messages        : 对话消息列表                           │
│  - currentTask     : 当前任务                               │
│  - uiState        : UI 状态（焦点、滚动）                   │
│  - toolResults    : 工具执行结果缓存                       │
│  - sessionHistory : 会话历史                                │
└─────────────────────────────────────────────────────────────┘
```

**重要说明**：
- `bootstrap/state.ts` **不是类**，是一个**模块**
- 导出 `STATE` 常量对象和多个存取函数
- 使用 `getInitialState()` 工厂函数创建初始状态

**状态流**：
```
用户操作 → Action → Reducer → 新状态 → UI 更新
                    │
                    ▼
              Middleware → 持久化
```

## 15.4 工程化实践

### 15.4.1 构建系统

Claude Code 使用 **Bun** 作为运行时和构建工具：

| 考量 | Bun 的优势 | Claude Code 的利用 |
|------|------------|-------------------|
| **启动速度** | 比 Node.js 快 3-4 倍 | CLI 快速启动 |
| **TypeScript** | 原生支持，无需编译 | 直接运行 .ts 文件 |
| **API 兼容** | 兼容 Node.js 模块 | 无缝使用 npm 包 |
| **内置工具** | bundler、package manager | 简化依赖管理 |

**构建配置特点**：
- 使用 `bun:bundle` 的 `feature()` 实现**条件编译**（功能开关）
- Dead Code Elimination (DCE) 移除未使用功能
- 支持 `KAIROS`、`DAEMON`、`VOICE_MODE`、`WEB_BROWSER_TOOL` 等特性标志

## 15.6 安全设计

### 15.6.1 权限系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      权限检查引擎                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   用户请求 ──→ 风险评估 ──→ 规则匹配 ──→ 决策               │
│                      │                                   │
│                      ▼                                   │
│              计算风险分数                                   │
│              0-100 范围                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**风险分类**：

| 级别 | 分数范围 | 默认行为 | 示例操作 |
|------|----------|----------|----------|
| **none** | 0 | 自动允许 | `ls`, `pwd` |
| **low** | 1-25 | 自动允许 | `npm install` |
| **medium** | 26-50 | 确认后执行 | `git commit` |
| **high** | 51-75 | 明确确认 | `rm -rf` |
| **critical** | 76-100 | 拒绝执行 | `rm -rf /` |

### 15.6.2 规则引擎

```typescript
// 规则匹配逻辑
const matchRule = (request: Request, rules: Rule[]): Decision => {
  // 1. 按优先级排序
  const sortedRules = rules.sort((a, b) => b.priority - a.priority)

  // 2. 找到第一条匹配的规则
  for (const rule of sortedRules) {
    if (matches(rule.condition, request)) {
      return rule.effect
    }
  }

  // 3. 默认规则
  return 'confirm'
}
```
