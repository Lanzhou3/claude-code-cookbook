# 第10章 权限系统深入

> 本章深入介绍 Claude Code 的权限系统，包括权限模型架构、规则配置、风险分类和自定义权限设置。

---

## 10.1 权限系统架构

### 10.1.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      权限检查引擎                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐   │
│   │   用户输入   │───→│  风险评估   │───→│   规则匹配   │   │
│   └─────────────┘    └─────────────┘    └─────────────┘   │
│                          │                   │            │
│                          ▼                   ▼            │
│                    ┌─────────────┐    ┌─────────────┐     │
│                    │  上下文收集 │    │   决策输出   │     │
│                    └─────────────┘    └─────────────┘     │
│                                             │            │
│                                             ▼            │
│                                      ┌─────────────┐     │
│                                      │  用户确认   │     │
│                                      └─────────────┘     │
└─────────────────────────────────────────────────────────────┘
```

### 10.1.2 权限检查流程

```
1. 用户发起请求
       │
       ▼
2. 收集请求上下文
   - 操作类型 (read/write/exec)
   - 目标路径
   - 命令参数
   - 当前环境
       │
       ▼
3. 风险评估
   - 查询风险数据库
   - 分析操作特征
   - 计算风险分数
       │
       ▼
4. 规则匹配
   - 检查 allow 规则
   - 检查 deny 规则
   - 应用优先级
       │
       ▼
5. 决策输出
   - allow (自动执行)
   - confirm (用户确认)
   - deny (拒绝执行)
```

### 10.1.3 核心组件

| 组件 | 职责 | 位置 |
|------|------|------|
| **PermissionManager** | 权限管理器 | 协调各组件 |
| **RiskClassifier** | 风险分类器 | 评估操作风险 |
| **RuleEngine** | 规则引擎 | 匹配权限规则 |
| **ConfirmationUI** | 确认界面 | 用户交互 |
| **AuditLogger** | 审计日志 | 记录所有操作 |

## 10.2 权限模式

### 10.2.1 四种权限模式

Claude Code 提供四种权限模式：

```json
{
  "permissions": {
    "mode": "default"
  }
}
```

| 模式 | 说明 | 确认频率 | 适用场景 |
|------|------|----------|----------|
| **default** | 标准模式 | 每次敏感操作 | 日常使用 |
| **plan** | 计划模式 | 只在计划阶段 | 自动化流程 |
| **acceptEdits** | 信任编辑 | 只对文件编辑 | 信任环境 |
| **auto** | 自动模式 | 不确认任何操作 | 完全自动化 |

### 10.2.2 模式选择建议

```
┌─────────────────────────────────────────────────────────────┐
│                    权限模式选择指南                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  初学者 → default (最安全，每次确认)                          │
│                                                             │
│  开发者 → acceptEdits (编辑自动通过，其他确认)                │
│                                                             │
│  自动化 → plan (计划阶段确认，执行不打扰)                      │
│                                                             │
│  高级 → auto (完全信任，不需要确认)                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 10.2.3 模式切换

```bash
# 命令行切换
claude --permissions plan

# 会话中切换
> /permissions acceptEdits

# 查看当前模式
> /permissions status

当前权限模式: default
状态: 活跃
```

## 10.3 规则配置

### 10.3.1 规则结构

```typescript
interface PermissionRule {
  id?: string;
  action: string | string[];
  resource?: string | Pattern;
  conditions?: Condition[];
  effect: 'allow' | 'deny' | 'confirm';
  priority?: number;
  reason?: string;
}
```

### 10.3.2 基本规则示例

```json
{
  "permissions": {
    "rules": [
      {
        "action": "file:read",
        "resource": "**/*.ts",
        "effect": "allow"
      },
      {
        "action": "file:write",
        "resource": "src/**",
        "effect": "allow"
      },
      {
        "action": "bash:exec",
        "command": "rm -rf *",
        "effect": "deny"
      }
    ]
  }
}
```

### 10.3.3 规则匹配顺序

```
1. 精确匹配 (exact match)
2. 通配符匹配 (wildcard)
3. 正则匹配 (regex)
4. 目录匹配 (directory)
5. 默认规则 (default)
```

## 10.4 风险分类

### 10.4.1 风险级别

| 级别 | 值 | 说明 | 默认行为 |
|------|------|------|----------|
| **none** | 0 | 无风险操作 | 自动允许 |
| **low** | 1-25 | 低风险操作 | 自动允许 |
| **medium** | 26-50 | 中等风险 | 确认后执行 |
| **high** | 51-75 | 高风险 | 明确确认 |
| **critical** | 76-100 | 极高风险 | 拒绝执行 |

### 10.4.2 操作风险评估

```bash
# 查看操作风险评估
> /risk bash:exec npm install
  命令: npm install
  风险级别: medium
  原因: 修改 node_modules

> /risk bash:exec rm -rf /
  命令: rm -rf /
  风险级别: critical
  原因: 递归删除根目录

> /risk file:write ~/.ssh/id_rsa
  文件: ~/.ssh/id_rsa
  风险级别: high
  原因: 敏感文件
```

### 10.4.3 自定义风险阈值

```json
{
  "permissions": {
    "riskThresholds": {
      "autoAllow": 20,
      "confirm": 50,
      "deny": 80
    }
  }
}
```

## 10.8 权限继承

### 10.8.1 项目级配置

```bash
# 项目配置优先级高于用户配置
.claude/settings.json
```

### 10.8.2 配置合并

```
用户配置 (~/.claude/settings.json)
        │
        ▼
    合并规则
        │
        ▼
项目配置 (.claude/settings.json)
        │
        ▼
    最终配置
```

### 10.8.3 配置覆盖

```json
// 用户配置
{
  "permissions": {
    "mode": "acceptEdits",
    "writablePaths": ["src/**"]
  }
}

// 项目配置
{
  "permissions": {
    "writablePaths": ["src/**", "tests/**"],
    "shell": {
      "deniedCommands": ["docker"]
    }
  }
}

// 最终配置 (合并后)
{
  "permissions": {
    "mode": "acceptEdits",
    "writablePaths": ["src/**", "tests/**"],
    "shell": {
      "deniedCommands": ["docker"]
    }
  }
}
```

## 10.9 审计日志

### 10.9.1 审计日志内容

```bash
# 查看审计日志
> /audit log

┌─────────────────────────────────────────────────────────────┐
│ 时间              │ 操作        │ 资源        │ 结果      │
├─────────────────────────────────────────────────────────────┤
│ 2024-04-01 10:00 │ file:read   │ src/a.ts    │ allow    │
│ 2024-04-01 10:01 │ file:write  │ src/b.ts    │ allow    │
│ 2024-04-01 10:02 │ bash:exec   │ rm -rf tmp  │ denied   │
│ 2024-04-01 10:03 │ file:read   │ ~/.ssh/id   │ denied   │
└─────────────────────────────────────────────────────────────┘
```

### 10.9.2 日志配置

```json
{
  "permissions": {
    "audit": {
      "enabled": true,
      "path": "~/.claude/audit.log",
      "maxSize": "10MB",
      "rotate": true,
      "includeAllow": false,
      "includeDeny": true
    }
  }
}
```

### 10.9.3 导出审计报告

```bash
# 导出 CSV
> /audit export --format csv > audit.csv

# 按日期范围导出
> /audit export --from 2024-01-01 --to 2024-03-31

# 统计报告
> /audit report --type summary
```
