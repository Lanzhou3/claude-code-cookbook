# 第12章 会话与记忆系统

> 本章详细介绍 Claude Code 的会话管理和记忆系统，包括会话生命周期、记忆机制、项目知识库和跨会话学习。

---

## 12.1 会话概述

### 12.1.1 什么是会话

会话（Session）是 Claude Code 中的基本工作单元，每次启动 CLI 到退出为一个会话。

```
会话生命周期
──────────────────────────────────────
启动 → 初始化 → 活跃期 → 保存 → 退出
         │         │
         ▼         ▼
      加载历史   处理请求
                   │
                   ▼
              更新上下文
──────────────────────────────────────
```

### 12.1.2 会话文件

```bash
# 会话文件位置
~/.claude/sessions/
├── 2024-04-01_10-00-00_abc123  # 活跃会话
├── 2024-04-01_09-30-00_def456  # 历史会话
└── 2024-03-31_16-45-00_ghi789  # 历史会话
```

### 12.1.3 会话元数据

```json
{
  "sessionId": "2024-04-01_10-00-00_abc123",
  "startTime": "2024-04-01T10:00:00Z",
  "lastActive": "2024-04-01T10:30:00Z",
  "model": "claude-sonnet-4-6",
  "messageCount": 45,
  "tokenUsage": 125000,
  "directory": "/home/user/project",
  "status": "active"
}
```

## 12.2 会话管理命令

### 12.2.1 基本命令

```bash
# 查看所有会话
> /sessions list

会话列表:
──────────────────────────────────────
[活跃] 2024-04-01_10-00-00  30分钟前  /home/user/project
[历史] 2024-04-01_09-00-00  2小时前    /home/user/other
[历史] 2024-03-31_16-00-00  昨天      /home/user/old
──────────────────────────────────────

# 切换会话
> /sessions switch 2024-04-01_09-00-00

# 查看当前会话
> /sessions current

会话 ID: 2024-04-01_10-00-00_abc123
状态: 活跃
运行时长: 30 分钟
消息数: 45
```

### 12.2.2 会话保存

```bash
# 手动保存
> /sessions save

# 保存到指定名称
> /sessions save my-feature-work

# 导出会话
> /sessions export --format json > session.json
```

### 12.2.3 会话恢复

```bash
# 恢复上次会话
> /resume

# 恢复指定会话
> /resume 2024-04-01_09-00-00

# 从文件恢复
> /resume --file /path/to/session.json
```

## 12.3 记忆系统架构

### 12.3.1 记忆分层

```
┌─────────────────────────────────────────────────────────────┐
│                    记忆系统架构                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐   │
│  │   短期记忆   │ ←→ │   中期记忆   │ ←→ │   长期记忆   │   │
│  │  (当前会话)  │    │ (最近会话)  │    │ (持久存储)  │   │
│  └─────────────┘    └─────────────┘    └─────────────┘   │
│         │                  │                  │           │
│         └──────────────────┼──────────────────┘           │
│                            ▼                              │
│                   ┌─────────────────┐                     │
│                   │   记忆索引      │                     │
│                   │   (快速检索)    │                     │
│                   └─────────────────┘                     │
└─────────────────────────────────────────────────────────────┘
```

### 12.3.2 记忆类型

| 类型 | 生命周期 | 内容 | 容量 |
|------|----------|------|------|
| **短期记忆** | 会话内 | 当前对话、临时状态 | 无限制 |
| **中期记忆** | 最近 7 天 | 最近项目上下文 | 100 个条目 |
| **长期记忆** | 永久 | 重要知识、项目概要 | 无限制 |

### 12.3.3 记忆存储

```bash
# 记忆存储位置
~/.claude/memory/
├── short-term/      # 短期记忆 (会话)
│   └── session_abc123.md
├── medium-term/     # 中期记忆 (最近)
│   └── recent/
├── long-term/       # 长期记忆 (持久)
│   ├── projects/
│   ├── skills/
│   └── knowledge/
└── index.md         # 记忆索引
```

## 12.4 记忆操作

### 12.4.1 保存记忆

```bash
# 保存当前上下文到记忆
> /remember 项目使用 Next.js 13 App Router

已保存到长期记忆:
──────────────────────────────────────
标题: 项目技术栈
内容: Next.js 13 App Router
标签: [tech-stack] [nextjs]
时间: 2024-04-01 10:30:00
──────────────────────────────────────

# 保存带标签
> /remember --tag project-config "TypeScript strict mode"

# 保存重要决策
> /remember --important "使用 Zustand 而非 Redux"
```

### 12.4.2 加载记忆

```bash
# 列出所有记忆
> /memory list

长期记忆 (5):
  1. [project-tech] 项目使用 Next.js 13
  2. [project-config] TypeScript strict mode
  3. [decision] 使用 Zustand 状态管理
  4. [api] 后端 API 在 /api 路由
  5. [style] 使用 Tailwind CSS

# 加载记忆到上下文
> /memory load project-tech

已加载: 项目使用 Next.js 13 App Router
```

### 12.4.3 搜索记忆

```bash
# 搜索记忆
> /memory search "Next.js"

搜索结果:
──────────────────────────────────────
1. [project-tech] 项目使用 Next.js 13
2. [api] Next.js API 路由配置
3. [deployment] Vercel 部署配置
──────────────────────────────────────

# 按标签搜索
> /memory search --tag project-config
```

## 12.5 项目知识库

### 12.5.1 项目记忆

```bash
# 创建项目知识库
> /project init

正在扫描项目...
✓ 检测到: React + TypeScript
✓ 检测到: Vite 构建
✓ 检测到: Tailwind CSS
✓ 检测到: ESLint + Prettier

项目知识库已创建: /home/user/project/.claude/memory.md
```

### 12.5.2 项目记忆内容

```markdown
# 项目知识库: my-app

## 基本信息
- 项目名称: my-app
- 技术栈: React 18 + TypeScript
- 构建工具: Vite 5
- 包管理器: npm

## 代码规范
- 使用 TypeScript strict mode
- 组件放在 src/components/
- Hooks 放在 src/hooks/
- 测试放在 src/__tests__/

## 常用命令
- npm run dev  - 开发服务器
- npm run build - 构建
- npm test     - 运行测试

## 架构说明
- 使用 React Query 管理服务端状态
- 使用 Zustand 管理客户端状态
- 组件采用原子化设计
```

### 12.5.3 项目记忆同步

```bash
# 手动同步
> /project sync

# 设置自动同步
> /project sync --auto

# 查看同步状态
> /project sync --status
```

## 12.6 跨会话学习

### 12.6.1 学习机制

```
会话 1                      会话 2
──────────────────────────────────────
用户: 使用 Zustand          用户: 状态管理问题
Claude: 推荐 Zustand        Claude: [加载记忆] 检测到 Zustand
    ↓                            ↓
保存: Zustand 决策           使用: Zustand 最佳实践
```

### 12.6.2 学习内容

| 类型 | 示例 | 学习时机 |
|------|------|----------|
| **技术决策** | 使用 Zustand | 用户确认 |
| **代码规范** | 组件放在 components/ | 多次观察 |
| **项目结构** | 目录组织方式 | 主动学习 |
| **偏好设置** | 喜欢使用 async | 统计推断 |

### 12.6.3 学习配置

```json
{
  "memory": {
    "learning": {
      "enabled": true,
      "confidenceThreshold": 0.8,
      "autoLearn": true,
      "learnFromErrors": true
    }
  }
}
```

## 12.7 MEMORY.md 文件

### 12.7.1 文件位置

```
~/.claude/MEMORY.md           # 用户级记忆
~/.claude/projects/xxx.md     # 项目级记忆
.claude/memory.md             # 项目内记忆 (git 管理)
```

### 12.7.2 文件格式

```markdown
---
name: my-project-memory
description: 我的项目记忆
type: project
---

# 项目记忆

## 技术栈
- React 18
- TypeScript
- Vite

## 项目结构
- src/components/ - 组件
- src/hooks/ - 自定义 Hooks
- src/utils/ - 工具函数

## 待办
- [ ] 配置 ESLint
- [ ] 添加单元测试
```

### 12.7.3 手动编辑

```bash
# 打开 MEMORY.md 编辑
> /memory edit

# 添加条目
> /memory add "使用 pnpm 作为包管理器"

# 删除条目
> /memory remove "使用 yarn"
```

## 12.9 隐私与安全

### 12.9.1 记忆加密

```json
{
  "memory": {
    "encryption": {
      "enabled": true,
      "key": "env:MEMORY_ENCRYPTION_KEY"
    }
  }
}
```

### 12.9.2 敏感信息处理

```bash
# 标记敏感记忆
> /remember --sensitive "API_KEY=xxx"

# 自动过滤
> /remember "用户的密码是 123456"
[警告] 检测到敏感信息，已过滤
```

### 12.9.3 记忆清除

```bash
# 清除所有记忆
> /memory clear --all

# 清除项目记忆
> /memory clear --project

# 清除特定条目
> /memory clear 记忆ID
```
