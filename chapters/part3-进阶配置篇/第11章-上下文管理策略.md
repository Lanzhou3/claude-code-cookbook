# 第11章 上下文管理策略

> 本章详细介绍 Claude Code 如何管理上下文窗口，优化 token 使用，以及实现智能上下文压缩。

---

## 11.1 上下文概述

### 11.1.1 什么是上下文

上下文是 Claude Code 在对话中使用的所有信息，包括：

```
上下文组成
├── 系统消息 (System)
│   - 内置工具定义
│   - 系统提示词
│   - 行为规则
├── 对话历史 (History)
│   - 用户消息
│   - AI 响应
│   - 工具调用
├── 项目上下文 (Project)
│   - 项目结构
│   - 配置文件
│   - 语言/框架检测
└── 当前请求 (Current)
    - 用户最新输入
    - 最近的工具结果
    - 临时状态
```

### 11.1.2 上下文窗口

Claude Code 的上下文窗口限制：

| 模型 | 上下文窗口 | Claude Code 保留 |
|------|------------|-----------------|
| Sonnet 4 | 200K tokens | ~160K 可用 |
| Opus 4 | 200K tokens | ~160K 可用 |
| Haiku 4 | 200K tokens | ~160K 可用 |

## 11.2 上下文分层

### 11.2.1 四层架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Layer 1: 系统级                           │
│  - 内置工具定义 (~2K tokens)                                 │
│  - 核心提示词 (~1K tokens)                                   │
│  - 固定不压缩                                               │
├─────────────────────────────────────────────────────────────┤
│                    Layer 2: 项目级                           │
│  - 项目配置 (几百 tokens)                                    │
│  - 技术栈检测                                               │
│  - 可按需精简                                               │
├─────────────────────────────────────────────────────────────┤
│                    Layer 3: 会话级                           │
│  - 对话历史 (可变)                                          │
│  - 可压缩/摘要                                             │
│  - 保留最近 N 轮                                            │
├─────────────────────────────────────────────────────────────┤
│                    Layer 4: 请求级                           │
│  - 当前输入 (~几百 tokens)                                  │
│  - 工具结果                                                │
│  - 临时状态                                                │
└─────────────────────────────────────────────────────────────┘
```

### 11.2.2 各层策略

| 层级 | 保留策略 | 压缩方式 |
|------|----------|----------|
| 系统级 | 完全保留 | 不压缩 |
| 项目级 | 按需加载 | 精简描述 |
| 会话级 | 最近优先 | 摘要+保留 |
| 请求级 | 全部保留 | 不压缩 |

## 11.3 消息压缩

### 11.3.1 自动压缩触发

```bash
> 触发压缩

当上下文达到以下阈值时自动压缩：
- 警告阈值: 70% (140K tokens)
- 压缩阈值: 85% (170K tokens)
- 紧急阈值: 95% (190K tokens)
```

### 11.3.2 压缩算法

Claude Code 使用多种压缩算法：

| 算法 | 说明 | 压缩比 |
|------|------|--------|
| **摘要** | 保留关键信息生成摘要 | 50-70% |
| **精简** | 删除冗余格式和空白 | 10-20% |
| **裁剪** | 截断长输出 | 可变 |
| **选择性保留** | 只保留关键结果 | 可变 |

### 11.3.3 压缩示例

```
压缩前 (原始对话):
──────────────────────────────────────
User: 帮我写一个函数处理用户注册

Assistant: [生成 100 行代码]

Tool: FileWriteTool - 写入 100 行

User: 添加参数验证
...
──────────────────────────────────────

压缩后 (摘要保留):
──────────────────────────────────────
[对话摘要] 用户请求创建用户注册函数，
Claude 生成了 100 行代码并写入文件，
用户后续要求添加参数验证。
[详细代码保留在工具结果中]
──────────────────────────────────────
```

## 11.4 智能上下文管理

### 11.4.1 相关性评分

Claude Code 评估每个上下文片段的相关性：

```typescript
interface ContextRelevance {
  source: string;        // 来源
  relevance: number;    // 0-1 分数
  tokens: number;        // token 数
  priority: 'keep' | 'compress' | 'drop';
}
```

### 11.4.2 上下文选择策略

```
输入: 用户请求 "重构 UserService 类"

上下文评分:
──────────────────────────────────────
src/UserService.ts     0.95  高相关 ✓ 保留
src/UserController.ts  0.72  中相关 ~ 压缩后保留
src/OrderService.ts    0.15  低相关 ✗ 摘要
tests/User.test.ts     0.65  中相关 ~ 压缩后保留
README.md              0.20  低相关 ✗ 摘要
──────────────────────────────────────
```

### 11.4.3 动态调整

```bash
# 根据请求类型调整上下文
> 重构 UserService 类

上下文策略:
- 加载 UserService 相关文件
- 保留测试文件
- 压缩不相关的业务逻辑文件
- 保留类型定义

> 解释整个项目架构

上下文策略:
- 扫描所有目录结构
- 加载 package.json
- 压缩所有源代码（只做概述）
- 保留配置文件
```

## 11.5 文件处理策略

### 11.5.1 大文件处理

| 文件大小 | 处理策略 |
|----------|----------|
| < 10KB | 完全加载 |
| 10-50KB | 加载 + 摘要 |
| 50-100KB | 部分加载 |
| > 100KB | 提示用户选择 |

### 11.5.2 文件截断

```
> 读取 large-file.ts

文件过大 (2.1MB)，采用以下策略：
──────────────────────────────────────
✓ 加载文件头 (1-100 行)
✓ 加载关键部分 (@export, @interface)
✓ 提示: 使用 /grep 搜索特定内容
✓ 提示: 使用 /read 2760-2800 查看特定行
──────────────────────────────────────
```

### 11.5.3 多文件策略

```bash
# 并行读取优化
> 读取 src/a.ts, src/b.ts, src/c.ts

读取策略:
- 并行读取 3 个文件
- 独立 token 计数
- 单独摘要压缩
- 合并到上下文
```
