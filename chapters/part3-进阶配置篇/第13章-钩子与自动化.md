# 第13章 钩子与自动化

> 本章详细介绍 Claude Code 的钩子（Hooks）系统，包括钩子配置、事件类型、命令钩子以及自动化工作流。

---

## 13.1 钩子系统概述

### 13.1.1 设计理念

Claude Code 的钩子系统是**基于配置的事件驱动系统**，而非传统的编程接口。钩子通过 JSON 配置定义，在特定事件发生时自动执行命令、提示词或 HTTP 请求。

```
┌─────────────────────────────────────────────────────────────┐
│                    钩子系统架构                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   配置源 ──→ 事件匹配 ──→ 执行钩子 ──→ 结果处理           │
│                                                             │
│   ┌─────────────────────────────────────────────────┐     │
│   │  settings.json 中的 hooks 配置                     │     │
│   │  {                                               │     │
│   │    "PreToolUse": [{                              │     │
│   │      "matcher": "Bash",                          │     │
│   │      "hooks": [{"type": "command", ...}]        │     │
│   │    }]                                            │     │
│   │  }                                               │     │
│   └─────────────────────────────────────────────────┘     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 13.1.2 核心概念

| 概念 | 说明 |
|------|------|
| **事件 (Event)** | 触发钩子的时机，如工具执行前/后 |
| **匹配器 (Matcher)** | 过滤条件，如工具名称模式 |
| **钩子命令** | 实际执行的命令，有 4 种类型 |
| **配置源** | 钩子配置来自用户设置、项目设置等 |

### 13.1.3 钩子类型

| 类型 | 说明 | 使用场景 |
|------|------|----------|
| `command` | 执行 Shell 命令 | 自动化脚本、通知 |
| `prompt` | LLM 评估 | 条件判断、内容处理 |
| `agent` | 代理执行 | 复杂验证任务 |
| `http` | HTTP 请求 | 外部 API 集成 |

## 13.2 钩子事件类型

### 13.2.1 完整事件列表

Claude Code 支持 **27 种钩子事件**（定义于 `entrypoints/sdk/coreSchemas.ts`）：

| 事件 | 触发时机 | 主要用途 |
|------|----------|----------|
| `PreToolUse` | 工具执行前 | 参数验证、修改输入 |
| `PostToolUse` | 工具执行后 | 结果处理、自动化 |
| `PostToolUseFailure` | 工具执行失败 | 错误处理、通知 |
| `PermissionRequest` | 权限请求时 | 权限审批流程 |
| `PermissionDenied` | 权限被拒绝 | 替代方案 |
| `Notification` | 发送通知 | 通知集成 |
| `UserPromptSubmit` | 用户提交提示词 | 内容过滤、预处理 |
| `SessionStart` | 会话启动 | 初始化操作 |
| `SessionEnd` | 会话结束 | 清理、保存 |
| `Stop` | 会话停止请求 | 优雅关闭 |
| `StopFailure` | 停止失败 | 错误处理 |
| `SubagentStart` | 子代理启动 | 监控、通知 |
| `SubagentStop` | 子代理停止 | 资源清理 |
| `PreCompact` | 上下文压缩前 | 准备压缩 |
| `PostCompact` | 上下文压缩后 | 验证结果 |
| `Setup` | 设置加载时 | 配置验证 |
| `TeammateIdle` | 队友空闲 | 任务分配 |
| `TaskCreated` | 任务创建 | 任务追踪 |
| `TaskCompleted` | 任务完成 | 后续处理 |
| `Elicitation` | 信息请求 | MCP 集成 |
| `ElicitationResult` | 信息响应 | 结果处理 |
| `ConfigChange` | 配置变更 | 配置同步 |
| `InstructionsLoaded` | 指令加载 | 指令管理 |
| `WorktreeCreate` | Worktree 创建 | 环境准备 |
| `WorktreeRemove` | Worktree 删除 | 资源清理 |
| `CwdChanged` | 工作目录变更 | 路径更新 |
| `FileChanged` | 文件变更 | 监控触发 |

### 13.2.2 常用事件详解

**PreToolUse - 工具执行前**

```json
{
  "matcher": "Bash",
  "hooks": [
    {
      "type": "command",
      "command": "echo 'Running: $ARGUMENTS'",
      "if": "Bash(git commit *)"
    }
  ]
}
```

**PostToolUse - 工具执行后**

```json
{
  "matcher": "Write",
  "hooks": [
    {
      "type": "command",
      "command": "npm run lint $ARGUMENTS"
    }
  ]
}
```

## 13.3 钩子配置

### 13.3.1 配置位置

钩子配置在 `settings.json` 的 `hooks` 字段中：

```json
// ~/.claude/settings.json (用户级)
// 或 .claude/settings.json (项目级)
{
  "hooks": {
    "PreToolUse": [...],
    "PostToolUse": [...],
    "SessionStart": [...]
  }
}
```

### 13.3.2 配置结构

```json
{
  "hooks": {
    "<事件名>": [
      {
        "matcher": "<匹配模式>",
        "hooks": [
          { "type": "command", ... },
          { "type": "prompt", ... },
          { "type": "agent", ... },
          { "type": "http", ... }
        ]
      }
    ]
  }
}
```

### 13.3.3 Matcher 模式

Matcher 用于过滤触发条件：

| Matcher 示例 | 匹配条件 |
|-------------|----------|
| `"Bash"` | Bash 工具 |
| `"Write"` | Write 工具 |
| `"Bash(git *)"` | Bash 工具且参数匹配 git 模式 |
| `"*(test)"` | 任何工具且参数包含 test |

## 13.4 钩子命令类型

### 13.4.1 Command 钩子

执行 Shell 命令：

```json
{
  "type": "command",
  "command": "echo 'Hello $ARGUMENTS'",
  "shell": "bash",
  "timeout": 5000,
  "if": "Bash(npm *)",
  "statusMessage": "Running npm hook..."
}
```

**可用变量**：

| 变量 | 说明 |
|------|------|
| `$ARGUMENTS` | 工具输入参数 |
| `$TOOL_NAME` | 工具名称 |
| `$SESSION_ID` | 会话 ID |
| `$CWD` | 当前工作目录 |

### 13.4.2 Prompt 钩子

使用 LLM 评估条件：

```json
{
  "type": "prompt",
  "prompt": "Is this safe to execute: $ARGUMENTS? Answer yes or no.",
  "model": "sonnet",
  "timeout": 10000
}
```

### 13.4.3 Agent 钩子

使用代理执行复杂任务：

```json
{
  "type": "agent",
  "prompt": "Review the code changes in $ARGUMENTS and report any issues.",
  "model": "opus",
  "timeout": 60000
}
```

### 13.4.4 HTTP 钩子

发送 HTTP 请求：

```json
{
  "type": "http",
  "url": "https://api.example.com/webhook",
  "method": "POST",
  "headers": {
    "Authorization": "Bearer $SECRET_TOKEN"
  },
  "body": "{\"event\": \"tool_use\", \"args\": \"$ARGUMENTS\"}",
  "timeout": 10000,
  "allowedEnvVars": ["SECRET_TOKEN"]
}
```

## 13.5 实际配置示例

### 13.5.1 Git Hooks

```json

  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",

          {

    ],

            "type": "command",

            "if": "Bash(git commit *)"
          }
        ]
      }
    ]
  }
}
```

## 13.6 钩子优先级

### 13.6.1 配置源优先级

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 最高 | 用户设置 | `~/.claude/settings.json` |
| ↓ | 项目设置 | `.claude/settings.json` |
| ↓ | 本地设置 | `.claude/settings.local.json` |
| ↓ | 策略设置 | `managed-settings.json` (企业) |
| ↓ | 插件钩子 | 插件提供的钩子 |
| 最低 | 内置钩子 | Claude Code 内置 |

### 13.6.2 退出码语义

| 退出码 | 含义 |
|--------|------|
| 0 | 成功，结果显示在 transcript |
| 2 | 阻塞错误，显示 stderr 给模型 |
| 其他 | 仅显示 stderr 给用户 |

## 13.7 函数钩子（Session-Only）

### 13.7.1 概述

函数钩子是在会话中临时定义的钩子，通过 SDK 或 CLI 设置：

```typescript
// 通过 SDK 设置
const session = await claude.sessions.create({
  plugins: [{
    hooks: {
      PreToolUse: [{
        matcher: "Bash",
        hooks: [{
          type: 'function',
          callback: (messages) => {
            console.log('Tool being used');
            return true; // 允许执行
          },
          errorMessage: 'Hook failed'
        }]
      }]
    }
  }]
});
```

### 13.7.2 函数钩子结构

```typescript
{
  type: 'function',
  id?: string,
  timeout?: number,
  callback: (messages: Message[], signal?: AbortSignal) => boolean | Promise<boolean>,
  errorMessage: string,
  statusMessage?: string
}
```
