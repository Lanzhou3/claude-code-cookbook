> 本章详细介绍 Claude Code 的完整配置选项，包括权限配置、模型设置、钩子配置和插件管理等。

## 14.1 配置系统概述

### 14.1.1 配置层次

```
┌─────────────────────────────────────────────────────────────┐
│                    配置优先级（高到低）                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 命令行参数 (--config, --model 等)                      │
│  2. 环境变量 (ANTHROPIC_*, CLAUDE_* 等)                    │
│  3. 项目配置 (.claude/settings.json)                        │
│  4. 用户配置 (~/.claude/settings.json)                      │
│  5. 默认配置 (内置默认值)                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 14.1.2 配置文件位置

| 配置 | 路径 | 范围 |
|------|------|------|
| **用户配置** | `~/.claude/settings.json` | 全局生效 |
| **项目配置** | `.claude/settings.json` | 当前项目 |
| **环境变量** | Shell 环境 | 会话级 |
| **命令行** | CLI 参数 | 当前命令 |

### 14.1.3 配置验证

```bash
# 验证配置
> /config validate

配置验证:
──────────────────────────────────────
✓ settings.json 格式正确
✓ apiKey 已设置
✓ model 默认值正确
⚠ permissions.mode 未设置，使用默认值
✓ hooks 配置正确
──────────────────────────────────────
配置状态: 有效
```

## 14.2 权限配置

### 14.2.1 权限模式

```json
{
  "permissions": {
    "defaultMode": "default",
    "allow": [],
    "deny": [],
    "ask": []
  }
}
```

**实际支持的权限模式：**
- `default` - 默认模式
- `plan` - 计划模式
- `acceptEdits` - 接受编辑模式
- `auto` - 自动模式（需要功能标志）
- `bypassPermissions` - 绕过权限模式

### 14.2.2 权限规则

```json
{
  "permissions": {
    "allow": [
      "Bash:pwd",
      "Bash:ls",
      "Read:*"
    ],
    "deny": [
      "Bash:rm -rf /",
      "Write:/etc/**"
    ],
    "ask": [
      "Bash:npm publish",
      "Write:~/.ssh/**"
    ]
  }
}
```

## 14.4 钩子配置

### 14.4.1 钩子基本配置

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "prompt",
            "prompt": "分析此命令是否安全: $ARGUMENTS。回答 safe 或 unsafe。",
            "model": "sonnet"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "npm run format -- $ARGUMENTS",
            "if": "Write(*.ts)"
          }
        ]
      }
    ]
  }
}
```

### 14.4.2 HTTP 钩子

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "http",
            "url": "https://api.example.com/hook",
            "method": "POST",
            "headers": {
              "Authorization": "Bearer $ENV_VAR"
            }
          }
        ]
      }
    ]
  }
}
```

### 14.4.3 允许的 HTTP 钩子目标

```json
{
  "allowedHttpHookUrls": [
    "https://hooks.example.com/*"
  ],
  "httpHookAllowedEnvVars": [
    "API_KEY",
    "WEBHOOK_SECRET"
  ]
}
```

    "git-helper@community": true

### 14.5.2 插件市场配置

```json
{
  "extraKnownMarketplaces": {
    "my-marketplace": {
      "source": {
        "source": "github",
        "repo": "owner/claude-plugins"
      },
      "autoUpdate": true
    }
  }
}
```

### 14.5.3 企业插件策略

```json
{
  "strictKnownMarketplaces": [
    {
      "source": "github",
      "repo": "approved/plugins"
    }
  ],
  "blockedMarketplaces": [
    {
      "source": "npm",
      "package": "@untrusted/plugin"
    }
  ]
}
```

## 14.6 MCP 服务器配置

### 14.6.1 允许/拒绝 MCP 服务器

```json
{
  "allowedMcpServers": [
    { "serverName": "filesystem" },
    { "serverCommand": ["npx", "@modelcontextprotocol/server-github"] },
    { "serverUrl": "https://*.company.com/*" }
  ],
  "deniedMcpServers": [
    { "serverName": "untrusted-server" }
  ]
}
```

### 14.6.2 项目 MCP 服务器

```json
{
  "enableAllProjectMcpServers": true,
  "enabledMcpjsonServers": ["allowed-server-1", "allowed-server-2"],
  "disabledMcpjsonServers": ["blocked-server"]
}
```
